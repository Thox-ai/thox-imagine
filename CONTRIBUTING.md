# Contributing to thox-imagine

Thank you for your interest in contributing to thox-imagine! This project is maintained by **THOX.ai LLC**.

## Code of Conduct

By participating in this project, you agree to maintain a respectful and collaborative environment.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **Make your changes** following the code style and conventions of the project.
3. **Write tests** for any new functionality.
4. **Run the test suite** to ensure everything passes.
5. **Submit a Pull Request** with a clear description of your changes.

## Pull Request Process

1. Ensure your code passes all CI checks.
2. Update documentation if you change functionality.
3. Add or update tests as needed.
4. Request review from a maintainer.
5. A maintainer will review and merge your PR once approved.

## Commit Message Format

We follow [Conventional Commits](https://www.conventionalcommits.org/):

```
type(scope): short description

Optional body. Wrap at 72 characters.
```

Types: `feat`, `fix`, `refactor`, `docs`, `test`, `ci`, `chore`, `perf`

## Signed-off-by

By submitting a pull request, you certify that:

1. The contribution is your original work
2. You have the right to submit it under the MIT License
3. You agree to the project's licensing terms

Include `Signed-off-by: Your Name <your.email@example.com>` in your commit message.

## Development Setup

```bash
git clone https://github.com/Thox-ai/thox-imagine.git
cd thox-imagine
# Follow installation instructions in README.md
```

## Code Style

- Follow existing code conventions in the repository
- Run linters before submitting (if configured)
- Keep functions small and focused
- Document public APIs
